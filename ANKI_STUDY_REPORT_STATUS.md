# Отчет по текущему состоянию Anki-расширения

## Название

Расширение: **Anki Study Report**

Папка проекта: `anki_study_report`

Назначение: read-only дополнение для Anki, которое строит учебный отчет по статистике повторений, новым карточкам, ошибкам, времени занятий, проблемным колодам и нагрузке на завтра.

## Совместимость

- Ориентировано на **Anki 26.05+**
- Использует **Python 3.13**
- UI сделан на **PyQt6**
- В `manifest.json` указан `min_point_version: 260500`
- Дополнение не изменяет карточки, колоды или историю повторений, только читает данные из коллекции Anki

## Текущий функционал

### 1. Встраивание в интерфейс Anki

Расширение добавляет пункт меню:

**Tools -> Study Report**

По нажатию открывается отдельное окно отчета.

### 2. Окно отчета

В интерфейсе есть:

- выбор периода;
- выбор области отчета;
- список колод с чекбоксами;
- опция включения дочерних колод;
- текстовое поле с результатом;
- кнопка построения отчета;
- кнопка копирования Markdown;
- кнопка сохранения Markdown в `.md`;
- кнопка закрытия.

### 3. Поддерживаемые периоды

Расширение умеет строить отчет за:

- сегодня;
- вчера;
- последние 7 дней;
- последние 30 дней;
- пользовательский период;
- все время.

Для пользовательского периода есть два поля даты: начало и конец.

Если выбран очень большой период, больше 180 дней, расширение показывает предупреждение, что расчет может занять заметное время.

### 4. Область отчета

Можно строить отчет:

- по всем колодам;
- по текущей колоде;
- по выбранным колодам.

Есть настройка **"Включать дочерние колоды"**.

Если она включена, расширение пытается включить дочерние колоды через структуру имен вида `Parent::Child`.

### 5. Сохраняемые настройки

Расширение сохраняет пользовательские настройки через Anki config:

- последний выбранный период;
- даты пользовательского периода;
- область отчета;
- выбранные колоды;
- включение/отключение дочерних колод.

Стартовый `config.json`:

```json
{
  "default_period": "today",
  "report_scope": "all",
  "selected_deck_ids": [],
  "include_child_decks": true
}
```

## Собираемые метрики

Расширение читает данные из `revlog` и `cards`.

Сейчас считаются:

- общее количество повторений;
- количество новых карточек;
- количество ответов `Again`;
- процент успешных ответов;
- оценочное время занятий;
- разбивка по колодам;
- количество карточек, запланированных на завтра.

### Время занятий

Время берется из `revlog.time`.

Есть защита от выбросов:

- отрицательное время считается как `0`;
- один ответ ограничивается максимумом **120 секунд**.

Это нужно, чтобы случайные зависшие или аномально длинные ответы не ломали отчет.

### Процент успешности

Формула:

```text
pass_rate = (total_reviews - again_count) / total_reviews
```

Если повторений нет, успешность считается `0`.

## Форматы отчета

Есть три шаблона:

### 1. Short

Короткий текстовый отчет на несколько строк.

### 2. Detailed

Подробный plain-text отчет с блоками:

- итоги;
- качество;
- проблемные колоды;
- завтра.

Именно он показывается в окне расширения.

### 3. Markdown

Markdown-отчет для копирования или сохранения в файл.

В Markdown входят разделы:

- `Итоги`;
- `Качество`;
- `Проблемные колоды`;
- `Завтра`.

## Аналитика качества

Расширение дает текстовую интерпретацию качества дня:

- если успешность >= 90%: хороший день;
- если успешность >= 80%: нормальный день, но ошибки заметны;
- если успешность < 80%: тяжелый день, лучше не брать много новых карточек;
- если повторений нет: данных за период нет.

## Проблемные колоды

Колода считается проблемной, если:

- в ней минимум 5 повторений;
- успешность ниже 80%.

В отчете показываются до 5 проблемных колод, отсортированных от худшей успешности к лучшей.

Для каждой проблемной колоды выводится:

- название колоды;
- процент успешности;
- количество `Again`;
- общее количество повторений.

## Нагрузка на завтра

Расширение считает карточки, которые должны быть повторены завтра.

Учитываются:

- review-карточки с `queue = 2`;
- learning/relearning-карточки с `queue in (1, 3)` и временем due в пределах завтрашнего дня.

Если нагрузка на завтра высокая относительно сегодняшних повторений, отчет предупреждает пользователя.

## Производительность и безопасность

### Read-only режим

Расширение только читает данные из базы Anki. Оно не меняет карточки, колоды, расписание или историю повторений.

### Кэширование

Есть короткий кэш отчета на **5 секунд**.

Если пользователь сразу копирует Markdown или сохраняет файл после построения отчета, метрики не пересчитываются заново.

### Обработка ошибок

Ошибки UI и расчета перехватываются.

Если что-то ломается:

- в Anki показывается понятное сообщение;
- подробности печатаются в консоль Anki через traceback.

## Файлы проекта

Основные файлы:

- `__init__.py` - UI, меню Anki, выбор периода/колод, сохранение настроек, запуск отчета;
- `metrics.py` - сбор статистики из базы Anki;
- `report_builder.py` - подготовка человекочитаемого текста из метрик;
- `templates.py` - русские шаблоны и пороги качества;
- `config.json` - настройки по умолчанию;
- `manifest.json` - метаданные Anki-дополнения;
- `anki_study_report.zip` - собранный архив расширения.

## Текущее состояние

Расширение уже имеет рабочий каркас и основную логику:

- добавляется в меню Anki;
- открывает окно;
- позволяет выбрать период и колоды;
- собирает статистику;
- строит подробный отчет;
- копирует Markdown;

- сохраняет пользовательские настройки.
