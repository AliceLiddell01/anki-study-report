## Результат

C1.5R.7 завершён. Комплексный продуктовый контур R1–R6 прошёл обязательные профильные, канонические проверки без Docker и чистые полные real-Anki gates приёмки. Продуктовая приёмка владельцем зафиксирована как выполненная.

C1.6 и C1.6B не реализовывались. C1.6 является следующим этапом и не начат. C1.6B остаётся условным этапом.

## Исправления, закрытые во время приёмки

Два несоответствия scope E2E-harness исправлены без изменения production-поведения или публичных payload:

1. `smoke-api.py` теперь выполняет контракт Inspection Profiles для режимов `cards` и `full`.
2. `restart-anki.sh` теперь выполняет детерминированное структурное изменение Inspection Profiles для режимов `cards` и `full`.

Регрессионное покрытие добавлено в `tests/test_docker_smoke_helpers.py`.

Локальный мост Docker в WSL не мог получить доступ к репозиториям пакетов во время сборки image. Поэтому image один раз собран через BuildKit с host networking; сам real-Anki E2E выполнялся через каноническую Compose-сеть runtime. Это было только обходное решение окружения, не изменившее код репозитория или scope runtime-тестов.
