## Воспроизведённый дефект и решение по renderer

Старый adapter полного предпросмотра отдавал предпочтение `browser=True`, поэтому payload представлял Browser Appearance вместо семантики вопроса и ответа reviewer. Затем Cards рендерил `frontHtml` и в Inspector, и в расширенном модальном диалоге. Компоновка предпросмотра также задавала минимальный scale, который мог превышать вписывание по ширине.

Anki 26.05 подтверждает путь reviewer через `Card.render_output(reload=True, browser=False)`. Явный compatibility-fallback — reviewer `question(reload=True, browser=False)` вместе с `answer()` без аргументов. Browser Appearance остаётся только частью компактной идентичности отображения.

## Реализованная семантика

- полный предпросмотр использует один нативный рендер reviewer для активной просматриваемой карточки;
- лицевая сторона и ответ или обратная сторона остаются раздельными санитизированными сторонами payload;
- Inspector рендерит лицевую сторону;
- расширенный диалог открывается на ответе или обратной стороне;
- отрендеренный Anki `{{FrontSide}}` сохраняется без изменений; frontend не объединяет стороны;
- лицевая сторона заменяет AV-маркеры вопроса;
- обратная сторона разрешает AV-маркеры вопроса и ответа;
- ссылки на media представлены дедуплицированным объединением обеих сторон;
- ошибки нативного рендера безопасно переходят на существующий санитизированный renderer шаблона;
- явные helpers доступности лицевой и обратной стороны не позволяют показывать лицевую сторону как ответ;
- строки очереди и компактная идентичность не изменены; открытие ответа переиспользует payload активного просмотра;
- масштабирование Inspector определяется шириной и использует ограниченную внутреннюю вертикальную прокрутку;
- расширенные ответы вписываются по ширине и используют естественную вертикальную прокрутку диалога;
- повторное измерение при загрузке изображений, шрифтов и resize сохраняется;
- общий доступный модальный диалог остаётся portal-based, переводит оболочку приложения в inert, удерживает фокус, закрывается по Escape и по возможности восстанавливает фокус;
- подписи RU/EN различают лицевую сторону и ответ.

## Ошибки и исправления

- Начальная диагностика преобразования нашла два одинаковых test fake нативного рендера; staging-преобразование было сделано точным для обоих.
- На чистом runner отсутствовали скопированные assets dashboard, из-за чего два неизменённых теста static fallback возвращали встроенную страницу; перед профильными Python-тестами assets были собраны без изменения server-кода.
- Первый тест компоновки показал, что длинное содержимое Inspector всё ещё вписывалось по высоте после корректного масштаба по ширине; алгоритм исправлен на вписывание по ширине с вертикальной прокруткой.
- Временная ошибка quoting преобразования исправлена до проверки.
- Первая попытка завершения пыталась добавить в staged-состояние несуществующий путь сгенерированных assets; по контракту assets остаются незакоммиченными, а перенос проверенной реализации был повторён со строгой обработкой ошибок Git.

## Безопасность и конфиденциальность

Browser Appearance не используется для полного предпросмотра. JavaScript карточки не выполняется. Не добавлены iframe, object/embed, удалённые загрузки, необработанные локальные пути или раскрытие токена dashboard. Allowlist sanitizer, проверка media, редактирование URL и изоляция Shadow DOM сохранены.
